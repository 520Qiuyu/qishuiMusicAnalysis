# 汽水音乐解析下载工具原型说明

## 原型定位

当前原型位于 `proto/index.html`，用于在正式搭建 `Vite + React + Ant Design` 项目前，先确认页面的信息架构、视觉方向、响应式布局和主要交互状态。

它不是最终工程代码，而是一个可直接打开预览的静态 HTML 原型。后续正式开发时，可以把页面拆分为 React 组件，并使用 Ant Design 承载表单、按钮、列表、提示、进度条等基础交互。

## 设计思想

整体视觉参考苹果设计规范，核心原则是简洁、轻量、层次清晰、操作聚焦。

页面采用浅灰背景作为基础，通过大圆角卡片、柔和阴影、半透明毛玻璃效果和留白建立空间层次。主色使用接近 Apple Blue 的 `#0071e3`，用于解析按钮、关键状态和主要视觉引导；成功状态使用 Apple Green 风格的绿色，突出“解析完成”和“可下载”的正反馈。

设计上避免复杂装饰，重点放在三个地方：

1. 顶部标题区明确产品能力，让用户知道这是一个音乐链接解析与下载工具。
2. 输入区作为最高优先级操作入口，按钮和输入框尺寸较大，适合桌面和移动端点击。
3. 结果区以音乐封面、歌曲信息、播放器和下载按钮为核心，模拟真实解析完成后的使用场景。

## 页面结构

原型主要分为以下区域：

### 顶部导航

顶部使用悬浮毛玻璃导航条，包含产品名称、历史入口和新建入口。

后续落地 React 时可以拆分为 `AppHeader` 组件，并将“历史”“新建”按钮接入真实功能。

### Hero 展示区

Hero 区展示产品主标题和描述，用来传达工具价值：将音乐分享链接解析为可播放、可下载的音乐资源。

右侧的装饰播放卡片用于增强音乐产品氛围，在移动端会隐藏，避免占用屏幕空间。

### 链接解析区

这是页面最重要的操作区域，包含：

- 音乐分享链接输入框
- 解析按钮
- 本地历史、移动端适配、单文件构建等能力提示

正式开发时建议使用 Ant Design 的 `Input`、`Button`、`Space`、`Tag` 或 `Flex` 来实现。

### 音乐结果卡片

结果卡片用于展示解析完成后的音乐信息：

- 歌曲封面
- 歌曲名称
- 歌手、专辑、音质等元信息
- 播放器视觉区域
- 下载按钮

正式功能中，这个区域应由解析接口返回的数据驱动。没有解析结果时，可以显示空状态或隐藏该区域。

### 下载管理

下载管理区展示当前下载任务，包括：

- 文件名
- 文件大小或下载状态
- 下载进度
- 暂停按钮
- 取消按钮

后续如果支持批量解析和批量下载，建议将该区域升级为队列列表。React 中可使用 Ant Design 的 `List`、`Progress`、`Button` 和 `Tooltip` 组合实现。

### 下载历史

下载历史用于展示最近保存过的音乐记录。

当前原型展示的是静态数据，正式开发时可以使用 `localStorage` 持久化，最多保留最近 50 条记录。后续也可以扩展清空历史、重新下载、复制链接等操作。

### 扩展能力区

底部预留了三个后续扩展方向：

- 批量解析队列
- 音乐资料管理
- 下载偏好设置

这部分用于提前规划功能扩展，不影响当前核心流程。

## 功能说明

### 当前原型已表达的功能

- 输入音乐分享链接
- 点击解析按钮
- 展示解析完成状态
- 展示音乐信息
- 展示播放器视觉
- 点击下载按钮
- 查看下载任务进度
- 查看下载历史
- 支持移动端和 PC 端布局

### 当前原型中的交互

原型中内置了少量 JavaScript：

- 点击“解析”会显示轻提示
- 点击“下载”会显示轻提示

这些交互仅用于验证页面反馈方式，不包含真实解析和下载逻辑。

## 响应式设计

原型针对 PC、平板和移动端做了布局适配。

PC 端采用宽屏布局：

- Hero 区左右分栏
- 音乐结果和侧边信息区并排展示
- 下载管理和下载历史放在右侧

移动端采用单列布局：

- 隐藏右侧装饰卡片
- 输入框和解析按钮上下排列
- 音乐结果卡片纵向展示
- 下载按钮占满宽度
- 扩展能力卡片改为单列

这样可以保证核心操作路径在小屏幕上仍然清晰：粘贴链接、解析、查看结果、下载。

## 后续 React + Ant Design 落地建议

正式项目建议使用以下组件拆分方式：

```text
src/
  App.jsx
  main.jsx
  styles/
    index.css
  components/
    AppHeader.jsx
    HeroSection.jsx
    ParseForm.jsx
    MusicResultCard.jsx
    DownloadManager.jsx
    HistoryList.jsx
    FeatureCards.jsx
```

建议状态拆分：

- `currentMusic`：当前解析出的音乐信息
- `isParsing`：解析 loading 状态
- `downloadTasks`：下载任务队列
- `historyList`：下载历史

Ant Design 可优先使用：

- `Button`
- `Input`
- `Card`
- `List`
- `Progress`
- `Tag`
- `message`
- `Spin`
- `Empty`
- `ConfigProvider`

为了贴近苹果风格，建议保留自定义 CSS 变量和整体布局样式，不完全依赖 Ant Design 默认视觉。Ant Design 负责基础交互组件，自定义样式负责页面气质和品牌表达。

## 单文件 HTML 打包目标

后续 Vite 项目构建时，目标是将本项目业务代码打包进一个 HTML 文件。

建议方案：

- 使用 `vite`
- 使用 `@vitejs/plugin-react`
- 使用 `vite-plugin-singlefile`
- 保留 Remix Icon CDN，或后续改为本地图标方案

最终期望产物：

```text
dist/
  index.html
```

其中业务 JS 和 CSS 会被内联到 `dist/index.html` 中，方便部署和分发。
